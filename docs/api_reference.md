# Python API Reference

Markdrop is not just a CLI tool; it provides a robust, strongly-typed internal Python API. Both the document extraction and AI generation features are designed to be imported cleanly into automated backend pipelines, Jupyter Notebooks, or enterprise RAG ingestion loops.

---

## 1. Class: `MarkDropConfig`
Holds the operational settings for the initial PDF parsing, image extraction, and HTML rendering phase.

```python
from markdrop import MarkDropConfig
import logging

config = MarkDropConfig(
    # How sharp the extracted pixel matrices are. Higher means larger files but better AI Vision input.
    image_resolution_scale=2.0,            
    
    # Customization for the interactive Web application viewer
    download_button_color='#444444',       
    
    # Internal module structure logging. Does not leak into your primary application logger.
    log_level=logging.INFO,                
    log_dir='logs',
    
    # Where temporary `.xlsx` files generated by `add_downloadable_tables` are dumped
    excel_dir='markdrop_excel_tables'
)
```

---

## 2. Function: `markdrop()`
The main entry point for phase 1 extraction logic. Evaluates a binary PDF and converts the unstructured text format into a structured Markdown graph with associated extracted images.

```python
from markdrop import markdrop

# You can pass both local file paths and https:// URLs securely
html_path = markdrop(
    input_doc_path="path/to/research_paper.pdf", 
    output_dir="/data/dump", 
    config=config
)

print(f"Generated Web Viewer at: {html_path}")
```
*   **Returns:** A `Path` object pointing to the newly generated `research_paper.html` string. Note that it simultaneously generates `research_paper.md` inside that same directory.

---

## 3. Function: `add_downloadable_tables()`
An optional augmentation to `markdrop()`. Evaluates the generated HTML from phase 1, extracts all `<table>` elements into `pandas` dataframes, builds standalone Excel documents containing them, and dynamically updates the HTML to feature UI download buttons.

```python
from markdrop import add_downloadable_tables

html_string_modified = add_downloadable_tables(
    html_path, 
    config
)
```

## 4. Class: `ProcessorConfig`
Holds the operational instructions defining how phase 2 (The AI reasoning loop) should act on an existing Markdown file.

```python
from markdrop import ProcessorConfig, AIProvider

config = ProcessorConfig(
    # Target file
    input_path="data_dumps/research_paper.md",      
    output_dir="data_dumps/ai_enriched",
    
    # Primary routing interface
    ai_provider=AIProvider.GEMINI,
    
    # Model parameters. All have sensible March 2026 defaults. 
    # Must match the chosen ai_provider prefix.
    gemini_model_name="gemini-3.1-flash-lite", 
    
    # Feature Toggles
    # When True, the ![alt text](images/123.jpg) syntax is DELETED entirely, replaced by AI text.
    # When False, the AI text is merely appended beneath the image syntax in the Markdown doc.
    remove_images=False,                   
    
    # True to parse and summarize tabular data.
    table_descriptions=True,
    
    # Fault Tolerance in cases of Cloud Model rate-limiting
    max_retries=3,
    retry_delay=2, 
)
```

## 5. Coroutine: `process_markdown()`

As of `v4.0.0`, Markdrop fundamentally changed its processing architecture to rely on Python's `asyncio` event loop. `process_markdown` is no longer a simple synchronous function call; it is a **coroutine**.

A coroutine allows Markdrop to read an input Markdown file, find 50 embedded images and 10 tables, and fire 60 simultaneous REST requests to an AI Vision Provider, awaiting their responses concurrently. 

To use `process_markdown` in a Python script, you must wrap it in an `async` loop.

```python
import asyncio
from markdrop import process_markdown, ProcessorConfig, AIProvider

config = ProcessorConfig(
    input_path="data_dumps/research_paper.md",
    output_dir="data_dumps/ai_enriched",
    ai_provider=AIProvider.OPENAI
)

async def build_pipeline():
    # Because process_markdown is async, it must be `await`ed.
    enriched_file = await process_markdown(config)
    print(f"Successfully digested AI models into: {enriched_file}")

# Launch the loop
asyncio.run(build_pipeline())
```

## 6. Helper: `setup_keys()`
A programmatic interface equivalent to the `markdrop setup` CLI command. It safely stores the current process's environment keys.

```python
from markdrop import setup_keys

# Manually trigger the interactive CLI prompt
setup_keys("gemini")
```

---

## 7. Helper: `generate_descriptions()`
A function exposing the underlying asynchronous `generate_response()` model routing interface across batches of files directly. Useful if you want to skip PDF parsing entirely and simply want an AI to analyze 50 local images against a unified prompt prompt.

```python
from markdrop import generate_descriptions

csv_output_path = generate_descriptions(
    input_path='/datasets/images/batch_1',
    output_dir='/datasets/analysis/',
    prompt="Identify the exact SKU serial number printed on the device. Provide the SKU, or say 'UNKNOWN'.",
    
    # You can pass an array of providers to benchmark outputs against the EXACT image and prompt in the resulting CSV
    llm_client=['gemini', 'anthropic', 'qwen', 'molmo'] 
)
```
